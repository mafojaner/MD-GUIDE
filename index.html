Markdown Syntax Guide

Everything you need to write clean, well-structured README files. Each section shows the raw syntax alongside its rendered output.

#

Headings

Use # symbols to create headings. The number of symbols sets the heading level (H1–H6).

Heading levels 1 – 4 syntax
Markdown
# Heading 1
## Heading 2
### Heading 3
#### Heading 4
Preview

Heading 1

Heading 2

Heading 3

Heading 4

✦ Always put a space between the # and the heading text. Many parsers require it.
¶

Paragraphs & Line Breaks

Separate paragraphs with a blank line. End a line with two spaces for a soft line break.

Paragraph separation syntax
Markdown
First paragraph text here.

Second paragraph starts after
a blank line.
Preview

First paragraph text here.

Second paragraph starts after a blank line.

**

Emphasis

Bold, italic, and strikethrough let you highlight text inline.

Bold, italic, strikethrough inline
Markdown
**Bold text**
*Italic text*
***Bold and italic***
~~Strikethrough~~
Preview

Bold text

Italic text

Bold and italic

Strikethrough

—

Lists

Unordered lists use -, *, or +. Ordered lists use numbers followed by a period.

Unordered list syntax
Markdown
- Item one
- Item two
  - Nested item
- Item three
Preview
  • Item one
  • Item two
    • Nested item
  • Item three
Ordered list syntax
Markdown
1. First step
2. Second step
3. Third step
Preview
  1. First step
  2. Second step
  3. Third step
>

Blockquotes

Prefix lines with > to create a blockquote. They can be nested.

Blockquote & nested syntax
Markdown
> This is a blockquote.
>
> > Nested quote.
Preview

This is a blockquote.

Nested quote.

Horizontal Rule

Three or more dashes, asterisks, or underscores on their own line create a divider.

Divider variants block
Markdown
---

***

___
Preview
[]

Links

Links use the pattern [label](url). You can add a tooltip with a quoted title.

Inline links inline
Markdown
[Visit GitHub](https://github.com)

[With tooltip](https://example.com "Hover me")
Preview

Visit GitHub

With tooltip

!

Images

Images follow the same pattern as links, prefixed with a !. The alt text improves accessibility.

Image embed syntax
Markdown
![Alt text](./image.png)

![With title](./img.png "Caption")
Preview
[ image: Alt text ]
` `

Code

Inline code uses single backticks. Fenced code blocks use triple backticks with an optional language tag for syntax highlighting.

Inline code inline
Markdown
Run `npm install` to start.
Preview

Run npm install to start.

Fenced code block block
Markdown
```javascript
const greet = (name) => {
  return `Hello, ${name}!`;
};
```
Preview
const greet = (name) => {
  return `Hello, ${name}!`;
};
|

Tables

Use pipes | to create columns. A separator row of dashes defines the header. Colons in the separator control alignment.

Basic table block
Markdown
| Name    | Role    |
| ------- | ------- |
| Alice   | Dev     |
| Bob     | Design  |
Preview
NameRole
AliceDev
BobDesign
☑

Task Lists

GitHub and many Markdown renderers support task lists with checkboxes. Use - [ ] for unchecked and - [x] for checked.

Checkbox list GFM
Markdown
- [x] Write the README
- [x] Add screenshots
- [ ] Publish to GitHub
Preview
  • ✅ Write the README
  • ✅ Add screenshots
  • ☐ Publish to GitHub
🏷

Badges

Badges are just Markdown images pointing to shield services like shields.io. They're common at the top of READMEs.
